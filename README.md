# samba-admin — Терминальная веб-панель для управления Samba

Лёгкая веб-панель в стиле терминала для создания Linux-пользователей, управления доступом к Samba-шарам и смены samba-паролей. Предназначена **только для локальной сети или VPN**.

## 🛡️ Архитектура и безопасность

1. **Принцип наименьших привилегий**: Flask-приложение работает от пользователя `www-panel` без прав root. Все привилегированные операции (`useradd`, `smbpasswd`, `net ads join`) выполняются через единственный скрипт `/usr/local/sbin/samba-admin-helper.sh` строго через `sudo` (настроено в `/etc/sudoers.d/samba-admin`).
2. **Безопасная передача паролей**: Пароли передаются в хелпер только через `stdin`, чтобы не отображаться в `ps aux`.
3. **Rate-limiting**: Экспоненциальная задержка при неудачных попытках входа (2с → 4с → ... → 5 мин). Реализована в памяти процесса для предотвращения DoS через блокировку IP.
4. **Production-окружение**: Используется `gunicorn` с 1 воркером и 4 потоками (`gthread`). Это гарантирует общую память для rate-limit и сессий, предотвращая обход лимитов при масштабировании воркеров. Таймаут увеличен до 120с для долгих операций (сканирование, `du`).
5. **Сетевая изоляция**: Flask слушает только `127.0.0.1:5000`. Внешний доступ только через `nginx` (порт 443) с самоподписанным или Let's Encrypt сертификатом. `ufw` открывает только 443 и 445 порты.

## ⚙️ Основные возможности

### 👤 Управление пользователями
- Создание, удаление (с домашней папкой), включение/выключение доступа к шарам (через группу `sharegroup`).
- Смена samba-пароля без изменения системного.
- Фильтрация и поиск по списку пользователей на клиенте.

### 📁 Управление шарами
Конфигурация хранится в `/etc/sambapanel/shares.db` и перегенерирует `/etc/samba/panel-shares.conf`. Основной `smb.conf` не модифицируется, используется `include`.
- **Создание/Удаление**: Создаёт папку, выставляет `chown`/`chmod 2775`. Удаление шары из панели **не удаляет** файлы с диска (защита от случайной потери данных).
- **Ограничение доступа**: `hosts allow` / `hosts deny` на уровне Samba (IP/подсети).
- **Veto files**: Запрет записи файлов по расширениям (например, `exe, bat, vbs`).
- **Корзина (`vfs recycle`)**: Перемещение удалённых файлов в `.recycle`. Поддержка автоочистки через `systemd` таймер (по возрасту файлов) или ручной очистки. Восстановление файлов с защитой от перезаписи (суффикс `.restored-<unix-time>`).
- **Антивирус (`vfs virusfilter` + ClamAV)**: Сканирование при открытии/закрытии файла. Заражённые файлы перемещаются в `.quarantine` с переименованием.
- **Мониторинг квот**: Отображение реального объёма (`du`) и процента занятости раздела. *(Для жёстких лимитов требуется ручная настройка XFS project quota).*
- **Бэкапы**: Ежедневный `tar.gz` через `samba-backup.timer` (с рандомизацией времени). Хранит последние N архивов в `BACKUP_DEST`. Восстановление **только через CLI** (`samba-restore.sh`) с превью и явным подтверждением для предотвращения случайной перезаписи.

### 🌐 Интеграция с Active Directory (Member Server)
Панель позволяет присоединить сервер к домену через `winbind` (сервер не становится контроллером домена).
- Автоматическая установка пакетов, настройка `/etc/krb5.conf`, `/etc/nsswitch.conf` и блока `[global]` в `smb.conf` (с маркерами `BEGIN/END AD INTEGRATION`).
- **Откат при ошибке**: Если `testparm` или `net ads join` завершаются с ошибкой, `smb.conf` восстанавливается из резервной копии.
- **Ограничения**: Не управляет системным hostname, не проверяет синхронизацию времени (требует `< 5 мин` расхождения) и не настраивает DNS-резолвинг на контроллер домена. AD-группы пока нельзя напрямую назначать владельцами шар через UI.

## 📊 Наблюдаемость и аудит

1. **Активные подключения**: Вывод сырого `smbstatus` (без парсинга) для точной сверки с системными логами.
2. **Диски и SMART**: Просмотр `lsblk`, монтирование разделов (с добавлением в `fstab` по UUID с опцией `nofail`). Просмотр статуса SMART (`PASSED`/`FAILED`/`UNSUPPORTED`). *Форматирование дисков через панель отключено из соображений безопасности.*
3. **Журнал действий панели**: `/var/log/sambapanel/audit.log` фиксирует все изменяющие операции (входы, создание/удаление, смена прав) с указанием IP и результата. Ротируется через `logrotate`.
4. **Полный аудит файлов (`vfs_full_audit`)**: Включается пошарово. Логирует все VFS-операции через `rsyslog` (facility `LOCAL5`) в `/var/log/samba-full-audit/full-audit.log`. Панель показывает последние 200 строк.

## 🚀 Установка

### Автоматическая (рекомендуется)
Скрипт идемпотентен, настраивает `gunicorn`, `nginx`, `ufw`, `clamav`, `logrotate` и запрашивает пароль администратора.
```bash
chmod +x install.sh
sudo ./install.sh
```

### Ручная (основные шаги)
```bash
# 1. Пользователь сервиса
sudo useradd -r -m -d /opt/sambapanel -s /usr/sbin/nologin www-panel

# 2. Развёртывание кода
sudo mkdir -p /opt/sambapanel
