Уведомления о падении сервисов

Если smbd, clamav-daemon, nginx или сама панель упадут — придёт сообщение сюда. Пустые поля = канал не используется, можно заполнить только один или оба сразу.

Почта требует настроенного ниже SMTP-релея — без него письма никуда не уйдут.

Настройка отправки почты (SMTP)

Нужно один раз указать данные твоего почтового ящика (через какой сервер уходят письма) — панель сама поставит и настроит msmtp. Пароль хранится в файле /etc/msmtprc с правами 600 (читает только root) — это требование самой программы msmtp, а не наша прихоть.

Присоединить сервер к домену

Присоединение как member server (через winbind), а не контроллер домена. Текущие локальные пользователи и шары не трогаются — это второй, независимый способ входа, поверх уже работающего. Пароль администратора домена нигде не сохраняется, используется только один раз для самого присоединения.

Перед присоединением убедись, что DNS этого сервера указывает на DNS-сервер AD (иначе домен не найдётся), и что время на сервере синхронизировано с контроллером домена (Kerberos не работает при рассинхронизации часов больше 5 минут).

Выйти из домена

Убирает AD-конфигурацию из smb.conf и снимает компьютерную учётку сервера в AD. Локальные пользователи и шары не затрагиваются. /etc/krb5.conf и запись winbind в /etc/nsswitch.conf намеренно не откатываются автоматически.

Обновление панели

Проверяет последний релиз на GitHub и сравнивает с версией, которая сейчас установлена.

жми «проверить обновления»

$ system ready. ожидание команд 

$ set_share_hosts

IP-адреса или подсети, которым разрешён доступ к этой шаре. Пустой список — доступ без ограничений по IP.

$ set_share_veto

Расширения файлов, которые нельзя будет записать в эту шару (Samba блокирует создание таких файлов на уровне протокола). Пустой список — без ограничений.

$ set_share_recycle

Удалённые через шару файлы будут перемещаться в скрытую папку .recycle вместо безвозвратного удаления.

$ set_share_quota

Квота мониторинговая — панель подсветит превышение в «Место на диске», но физически ничто не мешает записать больше (Samba не умеет ограничивать объём без реальных ФС-квот).

$ ls .recycle

Восстановить конкретный файл обратно в шару. Если по тому же пути уже что-то есть — восстановленный файл получит суффикс с датой, ничего не перезапишется.

загрузка...

$ mount

Устройство не будет отформатировано — монтируется как есть. Если файловой системы на нём нет, mount откажет с понятной ошибкой.

$ ls /srv

Двойной клик — зайти в папку. Шара будет создана как выбранная папка / имя шары — саму папку с именем шары создавать не нужно, панель сделает это сама.

$ set_share_group

Можно переключить между обычной локальной группой и доступом через Active Directory в любой момент, не пересоздавая шару.
